# Regra de Atualização de Steerings

## Quando Atualizar

Após concluir TODAS as user stories de um épico, verificar se algum steering precisa ser atualizado.

## Mapa de Impacto

| Mudança | Steerings Impactados |
|---------|---------------------|
| Nova dependência no pom.xml | architecture.md |
| Novo padrão de código adotado | code-standards.md |
| Nova pasta/módulo criado | architecture.md |
| Mudança em endpoints/rotas | architecture.md |
| Nova variável de ambiente | architecture.md |

## Regras

1. Atualizar APENAS seções relevantes — não reescrever o arquivo inteiro.
2. Manter consistência entre steerings.
3. Listar ao usuário quais steerings foram atualizados e o que mudou.
